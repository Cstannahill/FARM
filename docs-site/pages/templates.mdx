# Templates

Jumpstart your FARM Framework project with our professionally crafted templates. Each template is production-ready and includes comprehensive documentation, best practices, and deployment guides.

<Callout type="info">
  All templates are fully typed with TypeScript and include Docker
  configurations for seamless deployment.
</Callout>

## Quick Start

Get started with any template using the FARM CLI:

<CodeBlock language="bash">
  npx create-farm-app my-app --template [template-name] cd my-app npm run dev
</CodeBlock>

## Advanced Configuration

### Database Options

<CodeTabs defaultValue="mongodb">
  <CodeTab value="mongodb" label="MongoDB">
    <CodeBlock language="typescript">
      {`// farm.config.ts
export default defineConfig({
  database: {
    type: 'mongodb',
    url: process.env.MONGODB_URL || 'mongodb://localhost:27017/farmapp'
  }
});`}
    </CodeBlock>
  </CodeTab>
  <CodeTab value="postgresql" label="PostgreSQL">
    <CodeBlock language="typescript">
      {`// farm.config.ts
export default defineConfig({
  database: {
    type: 'postgresql',
    url: process.env.DATABASE_URL || 'postgresql://user:pass@localhost/farmapp'
  }
});`}
    </CodeBlock>
  </CodeTab>
</CodeTabs>

### AI Configuration

For AI-enabled templates, configure your AI providers:

<CodeBlock language="typescript">
  {`// farm.config.ts - AI Chat Template Example
export default defineConfig({
  template: 'ai-chat',
  ai: {
    providers: {
      ollama: {
        enabled: true,
        url: 'http://localhost:11434',
        models: ['llama3.1', 'codestral'],
        defaultModel: 'llama3.1',
        autoStart: true,
        autoPull: ['llama3.1']
      },
      openai: {
        enabled: true,
        apiKey: process.env.OPENAI_API_KEY,
        models: ['gpt-4', 'gpt-3.5-turbo'],
        defaultModel: 'gpt-3.5-turbo'
      }
    },
    routing: {
      development: 'ollama',
      production: 'openai'
    }
  }
});`}
</CodeBlock>

## Next Steps

After creating your project:

1. **Setup Environment**

   <CodeBlock language="bash">
     cp .env.example .env # Edit .env with your configuration
   </CodeBlock>

2. **Install Dependencies**

   <CodeBlock language="bash">npm install</CodeBlock>

3. **Start Development**

   <CodeBlock language="bash">npm run dev</CodeBlock>

4. **Deploy to Production**
   <CodeBlock language="bash">npm run build npm run deploy</CodeBlock>
